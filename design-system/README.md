# Design System

The design system is prepared externally and is not implemented in this task.

Frontend screen implementation is blocked until the design system is approved and added to the repository. Future frontend work must use the approved tokens, components, and interaction rules instead of creating separate screen-local styles.

Expected future materials include design tokens, typography, spacing, colors, buttons, fields, period controls, tables, metric cards, ABC/RFM/status badges, chart containers, loading states, error states, empty states, and Storybook or equivalent component documentation.
